Prompt

The exact prompt given to both models — zero-shot, single turn, no follow-ups.

Build a complete, self-contained single HTML file (no external libraries or CDN imports)
that implements an interactive A* pathfinding visualizer with the following requirements:

## Grid & Interaction
- Render a 30x20 grid of cells using an HTML5 Canvas element
- The user can left-click-drag to draw wall cells (impassable)
- The user can right-click-drag to erase walls
- There is always exactly one Start node (green) and one End node (red)
- The user can drag the Start and End nodes to reposition them
- Holding Shift while drawing sets "weighted" terrain cells (cost = 5, visually distinct, e.g. sand-colored)

## Algorithm
- Implement A* from scratch using a min-heap or equivalent priority queue (no shortcuts)
- Use Manhattan distance as the heuristic (no diagonal movement)
- During animation, visually distinguish: unexplored, in Open Set (frontier), in Closed Set (visited), and Final Path cells
- The final shortest path must be highlighted distinctly once found

## Controls UI (rendered in HTML below the canvas)
- "Visualize" button: starts the animated step-by-step execution
- "Speed" slider: controls animation speed from slow (1 step/frame) to fast (20 steps/frame)
- "Clear Path" button: resets only the algorithm state, keeping walls/weights intact
- "Clear All" button: resets the entire grid
- "Random Maze" button: generates a random maze using recursive division or similar algorithm

## Visual & UX requirements
- The UI must look polished and modern — use a dark theme with clear visual hierarchy
- Cell colors must be visually distinct and intuitive (not just random colors)
- Display a live stat panel showing: steps taken, cells visited, path length (in cost units), and elapsed time in ms
- If no path exists, show a clear user-facing message (not a console error)
- The canvas must be responsive to the container width

## Code quality requirements
- Use clean, modular JavaScript — separate concerns into logical sections or objects (grid, algorithm, renderer, UI)
- No global variable soup — use a single state object or module pattern
- Comments must explain the A* logic specifically, not just restate what the code does

Deliver only the complete HTML file content, nothing else.
